# 교차도메인 패턴 라이브러리 (Cross-Domain Pattern Library)

> 무한사고엔진 Phase 3(교차도메인 충돌)에서 참조하는 핵심 문서.
> 사용자의 문제를 추상화한 뒤, 이 라이브러리에서 이종 도메인의 해결 원리를 가져와 역이식한다.

---

## 사용법

```
1. 사용자 문제의 추상 패턴을 식별한다
   예: "고객이 떠난다" → 추상: "요소의 이탈"
   
2. 추상 패턴 인덱스(아래)에서 해당 패턴을 찾는다

3. 사용자 도메인을 제외한 2~3개 도메인의 해결 원리를 가져온다

4. 원래 도메인에 역이식하여 새로운 해결 후보를 생성한다
```

---

## 추상 패턴 인덱스

| 추상 패턴 | 해당하는 구체적 문제 예시 |
|-----------|------------------------|
| 요소의 이탈 | 고객 이탈, 직원 퇴사, 메모리 누수, 열 방출, 생태계 종 감소 |
| 병목/정체 | 서버 과부하, 교통 체증, 공정 대기, 의사결정 지연, 혈관 막힘 |
| 신호 대 잡음 | 데이터 노이즈, 시장 소음, 가짜 뉴스, 면역 과잉 반응, 센서 오류 |
| 균형 유지 | 수요공급, 체온 조절, 생태계 균형, 포트폴리오 밸런스, pH 조절 |
| 확장/스케일링 | 사업 확장, 도시 성장, 세포 분열, 네트워크 확장, 군대 편성 |
| 적응/진화 | 시장 변화, 기후 적응, 면역 학습, 알고리즘 개선, 전술 변경 |
| 연쇄 반응 | 바이럴 확산, 도미노 효과, 핵분열, 금융 위기, 정보 전파 |
| 자원 배분 | 예산 분배, 병력 배치, 에너지 분배, 시간 관리, 영양소 분배 |
| 탐색 vs 착취 | R&D vs 생산, 탐험 vs 정착, 학습 vs 실행, 다각화 vs 집중 |
| 복원력/회복 | 재난 복구, 면역 회복, 경제 회복, 생태 복원, 시스템 장애 복구 |

## 도메인별 핵심 해결 원리 (20개)

### 01. 생물학/생태학
| 원리 | 설명 | 적용 가능 추상 패턴 |
|------|------|-------------------|
| **공생 (Symbiosis)** | 서로 다른 종이 상호 이익을 주고받는 관계 | 자원 배분, 확장 |
| **면역 기억 (Immune Memory)** | 한번 만난 위협 패턴을 기억하여 재발 시 즉각 대응 | 적응, 신호/잡음 |
| **자연선택 (Natural Selection)** | 환경에 적합한 형질만 살아남아 세대를 거듭하며 최적화 | 적응, 탐색/착취 |
| **항상성 (Homeostasis)** | 외부 변화에도 내부 상태를 일정하게 유지하는 자동 조절 | 균형 유지 |
| **개체군 밀도 효과** | 개체 수가 환경 수용력(K)에 가까워지면 성장률이 감소 | 확장, 병목 |

### 02. 물리학
| 원리 | 설명 | 적용 가능 추상 패턴 |
|------|------|-------------------|
| **엔트로피 증가** | 폐쇄계에서 무질서도는 항상 증가. 질서 유지에는 에너지 투입 필요 | 균형, 복원력 |
| **공명 (Resonance)** | 외부 주파수가 고유 주파수와 일치하면 에너지가 극대화 | 연쇄 반응, 확장 |
| **최소 작용 원리** | 자연은 에너지를 최소로 쓰는 경로를 선택 | 자원 배분, 병목 |
| **상전이 (Phase Transition)** | 온도/압력이 임계점을 넘으면 물질 상태가 급변 | 연쇄 반응, 적응 |
| **중첩/간섭 (Superposition)** | 여러 파동이 합쳐져 강화되거나 상쇄 | 신호/잡음 |

### 03. 군사/전략
| 원리 | 설명 | 적용 가능 추상 패턴 |
|------|------|-------------------|
| **집중의 원칙** | 분산하지 말고 결정적 지점에 전력을 집중 | 자원 배분, 병목 |
| **기동전 (Maneuver Warfare)** | 정면 충돌 대신 적의 약점으로 우회하여 빠르게 돌파 | 병목, 적응 |
| **종심 방어 (Defense in Depth)** | 단일 방어선 대신 여러 겹의 방어를 설치하여 돌파 지연 | 복원력, 균형 |
| **기만과 은폐** | 적의 판단을 흐리게 만들어 의사결정을 지연시킨다 | 신호/잡음 |
| **OODA 루프** | 관찰→방향설정→결정→행동의 사이클을 적보다 빠르게 돌린다 | 적응, 탐색/착취 |

### 04. 경제학/금융
| 원리 | 설명 | 적용 가능 추상 패턴 |
|------|------|-------------------|
| **한계효용 체감** | 같은 것을 추가할수록 얻는 가치가 줄어든다 | 자원 배분, 균형 |
| **기회비용** | A를 선택하면 B를 포기하는 비용. 모든 결정에 숨은 가격이 있다 | 탐색/착취 |
| **모럴 해저드** | 위험을 감수하지 않아도 되면 무책임해진다 | 이탈, 균형 |
| **네트워크 효과** | 사용자가 늘수록 각 사용자의 가치가 증가 | 확장, 연쇄 반응 |
| **다각화** | 상관관계 낮은 자산에 분산 투자하여 리스크 감소 | 복원력, 균형 |

### 05. 의학/면역학
| 원리 | 설명 | 적용 가능 추상 패턴 |
|------|------|-------------------|
| **백신 원리** | 약화된 위협을 미리 경험시켜 면역을 형성 | 복원력, 적응 |
| **자가면역** | 면역 시스템이 자기 세포를 적으로 오인하여 공격 | 신호/잡음, 균형 |

## 역이식 가이드

도메인 패턴을 사용자 문제에 역이식할 때의 5단계:

```
1. 원본 원리의 핵심 메커니즘을 한 문장으로 추출
   예: "면역 기억" → "한번 경험한 패턴을 저장하여 재발 시 즉각 대응"

2. 사용자 도메인의 등가물을 매핑
   예: "패턴" = 트레이딩에서 차트 이벤트 시퀀스
       "저장" = N-gram 데이터베이스
       "즉각 대응" = 시그널 자동 발생

3. 이식 후보 문장 생성
   예: "이전에 관찰된 차트 이벤트 시퀀스를 저장하고,
        동일 패턴 재발 시 자동으로 시그널을 발생시킨다"

4. L1(물리법칙) 위반 체크
   → 위반 없음 → 후보 유지

5. 기존 접근 대비 차별성 검증
   → 기존에 N-gram은 있지만 "자동 시그널 발생"은 없었음 → 차별적
```
