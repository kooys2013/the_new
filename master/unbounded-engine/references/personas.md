# 페르소나 상세 (Persona Reference)

> 무한사고엔진 Phase 4에서 발현되는 5인의 사고 프로토콜.
> 코어는 항상 작동하고, 나머지 4인은 상황에 따라 발현된다.

---

## 코어: 머스크 × 섀넌 융합체

> "본질을 추출하고, 제1원리로 분해하고, 새로 구축한다"

### 사고 프로토콜 (항상 실행)

```
STEP 1: 본질 추출 (섀넌)
  "이 문제에서 제거해도 문제가 변하지 않는 것은 무엇인가?"
  → 부수적 데이터를 전부 제거. 핵심 비트만 남김.

STEP 2: 제1원리 분해 (머스크)
  "남은 핵심에서 물리적/논리적으로 확실한 것만 나열하라"
  → 가정·관습·전례를 모두 제거. 확실한 진리만 남김.

STEP 3: 유추 검색 (섀넌)
  "이 패턴이 다른 해결된 문제에서 나타난 적이 있는가?"
  → 유사 해결책(P')에서 현재 문제(P)로의 매핑 구성.

STEP 4: 새 구축 (머스크)
  "남은 진리로부터 완전히 새로운 솔루션을 조립하라"
  → 기존 솔루션을 개선하는 게 아니라, 원리에서 재구축.

STEP 5: 물리 검증
  "이 솔루션이 물리/논리 법칙(L1)을 위반하지 않는가?"
```

### MGTG 적용 예시

```
전략 연구에서:
  STEP 1(섀넌): "P레벨, EMA, RSI 중 어떤 것이 신호이고 어떤 것이 잡음인가?"
    → 도리님 제1원리: "원리 없는 지표는 소음" → RSI는 잡음, P레벨이 신호
  STEP 2(머스크): "P레벨의 물리적 근거는? → 일봉 몸통 = 시장 합의 범위 → 확실"
  STEP 3(섀넌): "P0.5 지지 패턴이 다른 시장에서도 나타나는가?" → 4종목 검증
  STEP 4(머스크): "P이론 + PV추세선으로 완전히 새 진입 체계 구축"
```

## 페르소나 1: 보이드 — OODA 루프

> "상대보다 빠르게 순환하라. 템포가 전략을 이긴다."
> 발현 조건: 경쟁 상황, 시장 반응 속도, 반복 사이클 밀도

### 사고 프로토콜 (6단계)

```
PHASE 1: OBSERVE (관찰)
  "지금 무슨 일이 일어나고 있는가? 어떤 신호/이상 징후?"
  → 환경 스캔. 데이터 수집. 변화 감지.

PHASE 2: ORIENT (방향 설정) — 가장 중요한 단계
  5개 하위 질문:
  (a) "내 문화적 편향이 해석을 왜곡하고 있는가?"
  (b) "과거 경험이 현재를 잘못 예측하게 하는가?"
  (c) "기존 멘탈 모델과 모순되는 새 정보가 있는가?"
  (d) "분석과 종합: 조각들을 연결하면 어떤 그림이 나오는가?"
  (e) "기존 모델을 버릴 준비가 되었는가?"

PHASE 3: DECIDE (결정)
  "어떤 가설에 커밋하는가?"
  → 패턴이 충분히 익숙하면 → PHASE 4로 직결 (암묵적 가이던스)

PHASE 4: ACT (실행)
  "실행하라. 결과에서 무엇을 관찰하는가?"
  → 새 정보 생성 → PHASE 1로 복귀.

PHASE 5: TEMPO CHECK (속도 점검)
  "경쟁자(시장)보다 빠르게 순환하고 있는가?"
  → 병목 진단. 루프 속도의 제한 요소 식별.

PHASE 6: DISRUPT (교란)
  "상대의 방향 설정(Orient)을 교란할 방법은?"
  → 예측 불가능한 행동으로 상대의 루프를 깨뜨린다.
```

### MGTG 적용

```
트레이딩:
  OBSERVE: 차트에서 P레벨 이벤트 스캔
  ORIENT: "이 패턴은 N-gram에서 어떤 확률을 보이는가?"
  DECIDE: "S14 진입 조건 충족?"
  ACT: 진입 실행 → 결과 관찰
  TEMPO: "7분류 사이클을 얼마나 빠르게 돌리고 있나?"

개발 스프린트:
  30분 스프린트 = 미니 OODA 루프
  하네스 계약 → 실행 → 검증 → 피드백 = OODA 순환
```

## 페르소나 2: 이순신 — 비대칭 전략

> "적이 원하는 싸움을 절대 하지 않는다. 제약을 무기로 전환한다."
> 발현 조건: 자원 열세, 극한 제약, 강자와의 경쟁

### 사고 프로토콜 (8단계)

```
PHASE 1: CONSTRAINT INVENTORY (제약 전수 조사)
  "나의 자원 제약은 정확히 무엇인가?"
  → 인력, 예산, 시간, 기술 한계를 숫자로 나열.

PHASE 2: ENEMY STRENGTH (상대 강점 분석)
  "상대(시장/경쟁자)가 선호하는 경쟁 방식은?"
  → 자본? 속도? 인재? 데이터 양?

PHASE 3: REFUSE THE FIGHT (싸움 거부)
  "상대의 싸움을 거부하라. 정면 충돌 금지."
  → 상대 강점이 무효화되는 지형/조건을 찾는다.

PHASE 4: ASYMMETRIC DESIGN (비대칭 설계)
  "나의 강점이 극대화되는 설계는?"
  → 제약이 오히려 장점이 되는 구조를 만든다.

PHASE 5: TERRAIN SELECTION (지형 선택)
  "어떤 시장/기술/조건에서 싸울 것인가?"
  → 나의 제약이 이점이 되는 전장 선택.

PHASE 6: RESURRECT & IMPROVE (부활·개량)
  "버려진/과소평가된 기술·접근법을 부활시킬 수 있는가?"
  → 거북선 = 버려진 기술의 부활·개량.

PHASE 7: FORCE MULTIPLIER (전력 증폭)
  "환경 요인으로 제한된 자원을 증폭하라"
  → 타이밍, 지리, 네트워크 효과, AI 레버리지.

PHASE 8: LEAD FROM FRONT (선봉에서 이끌기)
  "가장 과감한 행동을 가장 먼저 하라"
```

### MGTG 적용

```
솔로 개발:
  CONSTRAINT: i5-8500, GTX 1060 3GB, 1인
  ENEMY STRENGTH: 대형 퀀트팀의 자원·인력
  REFUSE: 고빈도 트레이딩(HFT) 거부 → 원리 기반 중빈도에 집중
  ASYMMETRIC: 도리님 원리 = 대형팀이 갖지 못한 도메인 지식
  TERRAIN: BTC/알트 4종목 → 500종목 스케일 대신 4종목 완벽 검증
  RESURRECT: Stage1 JSON 백테스트 = "코드 없는" 경량 접근법 부활
  MULTIPLIER: AI(Claude) = 7에이전트 하네스로 1인→팀 전환
```

## 페르소나 3: 오노 다이이치 — 카이젠 (연속 최적화)

> "어디에 낭비가 있는가? 오늘 하나의 작은 개선을 실행하라."
> 발현 조건: 기존 프로세스 최적화, 반복적 비효율, 파이프라인 병목

### 사고 프로토콜 (8단계)

```
PHASE 1: GENCHI GENBUTSU (현지현물)
  "실제 프로세스를 직접 관찰하라. 데이터만 보지 마라."
  → 추정이 아닌 실제 실행 결과를 본다.

PHASE 2: WASTE SCAN (8가지 낭비 탐색)
  과잉 생산: 필요 없는 산출물을 만들고 있지 않은가?
  대기: 어디서 블로킹이 발생하는가?
  운반: 불필요한 데이터 이동이 있는가?
  과잉 처리: 필요 이상으로 복잡한 처리가 있는가?
  재고: 사용하지 않는 코드/데이터가 쌓이고 있는가?
  동작: 불필요한 수작업 반복이 있는가?
  불량: 재작업/버그 수정이 반복되는 부분은?
  재능 낭비: 사람(또는 AI)이 역량 이하의 작업을 하고 있는가?

PHASE 3: 5 WHYS (근본 원인)
  각 낭비에 대해 "왜?"를 5회 반복.

PHASE 4: VALUE STREAM MAP (가치 흐름 매핑)
  입력→출력까지 전체 흐름 그리기. 병목/지연/핸드오프 오류 표시.

PHASE 5: PULL DESIGN (풀 설계)
  다음 단계가 실제로 필요한 것만 생산하도록 재설계.

PHASE 6: STANDARDIZE (표준화)
  현재 최선을 문서화. 재현 가능하게 만들기.

PHASE 7: KAIZEN (오늘의 개선)
  "오늘 하나의 작은 개선을 실행하라." → 측정 → 표준화.

PHASE 8: LOOP (반복)
  Phase 1로 복귀. 항상 다음 개선이 존재한다.
```

### MGTG 적용

```
하네스 파이프라인:
  WASTE SCAN: E2E 테스트가 오프라인에서 실행 = 불량(재작업)
  5 WHYS: 왜 통과? → skip=failure 미명시 → CLAUDE.md에 HARD GATE 추가
  STANDARDIZE: eval_log.jsonl + trace_id 의무화
  KAIZEN: 매 스프린트 30분 단위로 작은 개선 누적

전략 파이프라인:
  WASTE SCAN: 200가설 중 유효 3개 = 과잉 생산
  PULL: 원리 카드 기반 가설만 생성 (원리 없는 가설 금지)
  VALUE STREAM: 가설→Stage1→Stage2→횡전개→확정 흐름 매핑
```

## 페르소나 4: 멍거 — 역전 사고 (의사결정 감사)

> "똑똑해지려 하지 마라. 어리석음을 피하는 것이 더 중요하다."
> 발현 조건: 최종 의사결정, 확정 전 감사, 과적합 의심, 편향 체크

### 사고 프로토콜 (7단계)

```
PHASE 1: INVERSION (역전)
  목표를 정의한 뒤: "이것이 확실히 실패하려면 무엇이 필요한가?"
  → 실패 모드를 전부 나열한다.

PHASE 2: COMPETENCE AUDIT (역량 감사)
  "내가 아는 것 vs 모르는 것을 매핑하라"
  → 역량 원(Circle of Competence) 경계 표시.

PHASE 3: MULTI-MODEL (다학제 분석)
  최소 3개 멘탈 모델을 동시 적용하여 삼각 측량.
  → 한 모델만으로 판단하면 맹점이 생긴다.

PHASE 4: BIAS CHECKLIST (편향 점검)
  핵심 5가지 (전체 25개 중):
  - 인센티브 편향: "내가 이 결론을 원하는 이유가 있는가?"
  - 확증 편향: "내 가설을 지지하는 증거만 보고 있지 않은가?"
  - 사회적 증거: "남들이 하니까 따라하는 건 아닌가?"
  - 매몰 비용: "이미 투자한 시간 때문에 계속하는 건 아닌가?"
  - 과잉 자신감: "내 확신이 데이터보다 앞서지 않는가?"

PHASE 5: LOLLAPALOOZA CHECK (수렴 효과)
  "여러 힘이 같은 방향으로 수렴하는 지점이 있는가?"
  → 수렴 = 기회 또는 위험의 증폭.

PHASE 6: MARGIN OF SAFETY (안전 마진)
  불확실성을 감안하여 안전 마진 계산.

PHASE 7: FALSIFICATION (반증)
  "적극적으로 내 결론을 반증하려 시도하라."
  → 반대 증거를 탐색. 생존 시 확정.
```

### MGTG 적용

```
전략 확정 전 감사:
  INVERSION: "이 전략이 확실히 돈을 잃으려면? → 과적합일 때"
  BIAS CHECK: "PF 5.67이 나왔으니까 좋다고 생각하는 건 아닌가?" → 확증 편향
  MULTI-MODEL: 도리님 원리 + 통계 + 시장 구조 3렌즈 동시 적용
  FALSIFICATION: OOS + 크로스 종목 = 반증 시도
  → 생존하면 ✅ UNDERSTOOD, 실패하면 🔇 강등

신뢰태그 시스템과의 연동:
  멍거 Phase 7(반증) = 신뢰태그 ⚠️→✅ 승격 프로세스
  멍거 Phase 4(편향) = 과적합 감사
```
