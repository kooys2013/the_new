# 공지·안내 메일 생성기 (Announcement Mail Generator)

## 개요

사내 공지·안내 이메일을 아래 표준 구조에 따라 생성한다.  
기본은 **한국어 단독**이며, 사용자가 일본어 병기를 요청하면 **한/일 이중 언어** 버전으로 출력한다.

---

## 표준 이메일 구조

```
[① 인사말 + 소개]
[② 배경/컨텍스트]
[③ 추진/도입 배경 (번호 목록 3~5개)]
[④ 핵심 요청 사항 (날짜 + 구체적 행동)]
[⑤ ※ 주의사항 / 참고사항]
[⑥ 문의처 + 의견 수렴 채널]
[⑦ 마무리 인사]
[⑧ 서명 블록]
```

---

## 섹션별 작성 가이드

### ① 인사말 + 소개
- 첫 줄: `수고많으십니다.` (또는 상황에 맞는 인사)
- 둘째 줄: `[소속팀] [이름]입니다.`
- 간결하게 1~2줄로 마무리

**예시**
```
수고많으십니다.
경영혁신팀 홍길동입니다.
```

---

### ② 배경/컨텍스트
- 최근 활동(회의, 프로젝트, 기간 등)을 1~2문장으로 요약
- 「[시스템명/활동명]」처럼 꺾쇠 인용부호 활용

**예시**
```
지난 ○월간 관련 부서와 협의를 통해 개발한 「○○시스템」의 도입을 안내드립니다.
```

---

### ③ 추진/도입 배경 (번호 목록)
- 핵심 목적 3~5개를 번호 목록으로 정리
- 각 항목은 **기대효과** 중심으로 작성
- 마지막에 `등을 기대하고 있습니다.` 로 마무리

**예시**
```
금번 ○○시스템 도입 추진 배경은 하기와 같습니다.
1. 기존 다수 시스템 통합을 통한 업무 효율화
2. 휴먼에러 감소 및 데이터 정합성 향상
3. 프로세스 표준화 및 관련 부서 간 연계 강화
등을 기대하고 있습니다.
```

---

### ④ 핵심 요청 사항
- 배포/시행 일시를 명확히 기재 (`○/○(요일) ○시`)
- 요청 행동을 구체적으로 서술 (테스트, 피드백, 참석 등)
- 기간이 있으면 범위 명시 (`○/○ ~ ○/○, 약 ○주간`)
- Tab 들여쓰기로 세부 일정 표기

**예시**
```
이에 ○월 ○일(월)부터 ○○시스템을 정식 배포하여,
생산기술 멤버분들의 Feedback을 적극 수렴하고자 합니다.
	배포 시점: ○/○ 오전 ○시 (사용 매뉴얼 동시 배포 예정)

○/○ ~ ○/○(약 ○주간) 1~2회 정도의 시스템 테스트를 요청드립니다.
구체적으로는 ○○ 기능을 1~2회 직접 사용해 주시기 바랍니다.
```

---

### ⑤ ※ 주의사항 / 참고사항
- `※` 기호로 시작
- 예외 처리, 기존 방식 유지 안내, 테스트 기간 주의사항 등 포함

**예시**
```
※ 현재는 테스트 기간인 관계로, 실제 업무는 기존 시스템을 통해 진행 부탁드립니다.
※ 오류 발견 시 하단 연락처로 즉시 공유해 주시면 감사하겠습니다.
```

---

### ⑥ 문의처 + 의견 수렴 채널
- 담당자 연락처 (전화, Teams, E-mail 등)
- 의견 수렴 방법 및 마감 일정 (Naver Form / MS Form 등)
- 미수신자 공유 요청

**예시**
```
궁금한 점이 있으신 분은 010-XXXX-XXXX 또는 Teams / E-mail로 연락주시기 바랍니다.
의견 수렴은 ○월 말 MS Form을 통해 진행할 예정입니다.
수신처에 없는 멤버분이 있으신 경우, 별도 공유 부탁드립니다.
```

---

### ⑦ 마무리 인사
```
이상입니다.
수고하십시오.
```

---

### ⑧ 서명 블록
```
──────────────────────────────
[이름 (영문명)]
[회사명]
[소속팀] [파트]
Mail) xxxx@company.com   Mobile) +82-10-XXXX-XXXX
──────────────────────────────
```

## 한/일 이중 언어 버전 규칙

사용자가 일본어 병기를 요청하거나, 수신자가 일본인을 포함할 경우 적용:

1. 한국어 전문 → `=====` 구분선 → 일본어 전문 순으로 출력
2. 일본어 인사: `いつもお世話になっております。` 로 시작
3. 한국어 번호 목록 → 일본어도 동일 번호 목록 유지
4. 서명 블록은 동일하게 반복
5. ※ 주의사항도 일본어로 완전히 번역

```
==============================================
```

## 작성 시 준수 사항

| 항목 | 기준 |
|------|------|
| 톤/경어 | 공손하되 과도한 겸양 지양, 자연스러운 사내 비즈니스 어체 |
| 날짜 표기 | `○/○(요일)` 형식 우선, 필요 시 월·일 한글 병기 |
| 시스템/제품명 | 「 」 꺾쇠 인용 부호 사용 |
| 목록 구분 | 추진배경은 번호 목록, 주의사항은 ※ 기호 |
| 강조 | 배포일, 테스트 기간 등 핵심 일정은 명확히 단독 줄 처리 |
| 분량 | 본문 200~400자 (일어 병기 시 2배) 권장 |

## 정보 수집 체크리스트

스킬 실행 전, 아래 항목 중 불명확한 것은 사용자에게 역질문하라 (최대 3개):

- [ ] **발신자 정보**: 소속팀, 이름, 연락처
- [ ] **공지 주제**: 시스템명, 활동명 등
- [ ] **추진 배경**: 도입/공지 이유 (없으면 AI가 상황에 맞게 생성)
- [ ] **요청 행동**: 구체적으로 무엇을 해달라는지
- [ ] **일정**: 배포일, 테스트 기간 등
- [ ] **언어**: 한국어 단독 or 한/일 이중 언어

불명확한 항목이 3개 이상이면 먼저 질문하고, 그렇지 않으면 합리적으로 추정하여 즉시 초안을 생성한다.
