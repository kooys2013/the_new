# 견적사양서 생성 스킬

## 개요

사용자가 제공한 공사 정보를 바탕으로 **표준 견적사양서 본문을 텍스트로 출력**한다.
워드(.docx) 파일 생성은 사용자가 **"특별히"**, **"워드로"**, **"docx로"**, **"파일로 만들어줘"** 등
명시적으로 파일 생성을 요청한 경우에만 수행한다.

---

## 1. 출력 기본 원칙

- **기본 출력**: 마크다운 텍스트 (채팅 화면에 바로 표시)
- **파일 생성 조건**: "특별히 파일로", "워드로 만들어줘", "docx 파일로" 등 명시적 요청 시에만
- **언어**: 한국어 기본, 일본어 병기 필요 시 명시
- **폰트 기준** (docx 생성 시 적용):
  - 전체 폰트: 맑은 고딕
  - 대제목: 13pt Bold
  - 소제목: 11pt Bold
  - 본문: 10.5pt

---

## 2. 출력 구조

견적사양서는 아래 순서로 구성한다.

### [출력 앞부분 — 항상 먼저 출력]

```
【검토 사항】

a. 공사 간 주요하게 검토해야 할 사항:
   - (공사 내용 분석 후 핵심 주의사항 자동 도출)
   - (예: 고소작업 안전관리, 인허가 선행 필요 여부, 동시작업 가부 등)

b. 추가로 확인해봐야 할 사항:
   - (예: 드레인 유도 위치, 하부 배관 간섭, 재질 결정 필요 여부 등)
```

> 이 섹션은 사용자가 제공한 정보 및 공사 내용을 분석하여
> Claude가 능동적으로 도출한 검토 포인트를 기재한다.
> 단순 복사가 아니라 공사 특성에 맞는 실질적 내용을 작성할 것.

## 3. 텍스트 출력 포맷 규칙

- 섹션 구분: 마크다운 `##` 또는 `###` 사용
- 항목 번호: `1.` `1)` 형식 유지
- 불릿 기호: `•` 사용 (마크다운 `-` 대신)
- 자재 사양: 품목명, 규격, 수량을 한 줄에 명확히 표기
- 검토 사항(a/b)은 반드시 본문 **앞**에 출력

## 5. docx 파일 생성 (명시적 요청 시에만)

사용자가 "특별히", "워드로", "파일로 만들어줘" 등을 요청한 경우에만 실행한다.

### 폰트·스타일 기준
```python
from docx import Document
from docx.shared import Pt, RGBColor
from docx.oxml.ns import qn

doc = Document()

# 기본 폰트 설정 (맑은 고딕)
style = doc.styles['Normal']
style.font.name = '맑은 고딕'
style.element.rPr.rFonts.set(qn('w:eastAsia'), '맑은 고딕')
style.font.size = Pt(10.5)

# 대제목: 13pt Bold
# 소제목: 11pt Bold
# 본문: 10.5pt
```

### 파일명 규칙
```
{KST yymmddhhmm}_견적사양서_{건명 요약}.docx
예) 2603181600_견적사양서_H1H2동대기방지시설안전개선.docx
```

### docx 생성 시 주의사항
- 이미지(현장 사진 등)는 삽입하지 않음 (워드 파일에 이미지 자동 삽입 불가)
- 텍스트 본문만 구성
- 검토 사항(a/b)도 문서 상단에 포함

## 6. 처리 흐름

```
1. 사용자 입력 분석
   → 공사 정보 추출 (이미지/첨부파일 포함 시 내용 반영)

2. 검토 사항 도출 (자동)
   → a. 주요 검토사항
   → b. 추가 확인사항

3. 견적사양서 텍스트 출력 (기본)
   → 검토 사항 → 본문 순서

4. (조건) docx 파일 생성
   → "특별히" / "파일로" / "워드로" 등 명시 요청 시에만
```

### 공사 완료 조건 (공사 유형별)
- 펜스·구조물: 흔들림 없이 고정, 도어 개폐 원활

- 전기 배선: 절연 저항 측정, 운전 확인
- 콘크리트 기초: 양생 완료, 강도 확인
