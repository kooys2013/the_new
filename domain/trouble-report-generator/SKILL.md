# Trouble Report Generator Skill

## 개요

KORENO / Nitto 계열 현장용 트러블(설비 고장·이상·사고) 보고서 PPT를 자동 생성한다.
언어는 실행 시 사용자에게 선택받는다 (한글 / 日本語).

---

## STEP 0: 언어 선택 및 정보 수집

### 0-1. 언어 선택
사용자에게 반드시 먼저 물어본다:

```
보고서 언어를 선택해 주세요 / 報告書の言語を選択してください:
1. 한글 (Korean)
2. 日本語 (Japanese)
```

### 0-2. 입력 정보 체크리스트

사용자가 제공한 정보에서 아래를 추출한다. 누락된 항목은 [미입력] / [未入力]로 처리:

| 필드 | 한글 레이블 | 日本語ラベル |
|------|------------|--------------|
| 제목 | 트러블 개요 제목 | トラブル概要タイトル |
| 발생일시 | 발생일시 | 発生日時 |
| 발생상황 | 발생상황 | 発生状況 |
| 원인 | 원인 | 原因 |
| 조치 | 조치 | 措置・対応 |
| 대책 | 대책 | 恒久対策 |
| 피해상황 | 피해상황 | 被害状況 |

## STEP 1: 텍스트 리뷰 출력 (필수 — PPT 생성 전)

### ⚠️ PPT 생성 금지 규칙 (HARD RULE)

**사용자가 명시적으로 OK/확인/진행/네 등의 컨펌을 보내기 전까지는 절대 PPT를 생성하지 않는다.**

- 사용자가 내용을 제공했더라도 → 텍스트 리뷰만 출력하고 대기
- 사용자가 수정 요청을 하면 → 텍스트 리뷰를 업데이트하고 다시 컨펌 대기
- "만들어줘" / "generate" 등의 표현이 처음 요청에 포함되어 있어도 → 텍스트 리뷰 먼저 출력, 컨펌 후 생성
- 컨펌 없이 PPT를 생성하는 것은 스킬 위반이다

컨펌 판단 기준:
- ✅ 컨펌으로 인정: "OK", "확인", "진행해", "그대로", "만들어", "네", "응", "ㅇㅇ", "고", "좋아", "진행"
- ❌ 컨펌 아님: 수정 요청, 추가 정보 제공, 질문

---

PPT를 생성하기 전, 먼저 추출된 내용을 아래 형식으로 텍스트로 출력하여 사용자가 내용을 검토할 수 있게 한다.

각 슬라이드 헤더에는 반드시 **요약 타이틀**을 포함한다.
요약 타이틀이란 해당 슬라이드의 핵심 메시지를 1~2줄로 압축한 문장이다.
예: `[요약 타이틀] 버너#1 점화플러그 이상으로 RTO 실화 발생 → 124분 생산 중단`

### 출력 형식 (한글)

```
═══════════════════════════════════════════
  [SLIDE 1] 트러블 개요
  [요약 타이틀] {이 슬라이드의 핵심 메시지 1~2줄 — Claude가 내용을 분석해 자동 작성}
═══════════════════════════════════════════

제  목: {제목}

발생일시: {발생일시}

발생상황:
  {발생상황 — 줄바꿈 포함 전체}

원  인: {원인}

조  치:
  {조치 — 타임라인 형태로 줄별 표시}

대  책:
  {대책}

피해상황:
  {피해상황}

═══════════════════════════════════════════
  [SLIDE 2] 계통도 및 현장 사진
  [요약 타이틀] {누액 발생 위치 및 설비 구성 — 계통도·현장 사진으로 설명}
═══════════════════════════════════════════

  → 사용자 제공 이미지/스크린샷을 배치합니다.
     현재 제공된 이미지: {이미지 목록 or "없음 — 사용자 제공 필요"}
     권장 구성: 좌측 계통도(HMI 화면 등) + 우측 현장 사진

═══════════════════════════════════════════
  [SLIDE 3] 5WHY 분석
  [요약 타이틀] {근본원인을 한 줄로 요약 — Claude가 5WHY 결과에서 도출}
═══════════════════════════════════════════

문제 현상: {원인의 최상위 현상}

Why 1: {현상 → 첫 번째 왜?}
Why 2: {Why 1 → 두 번째 왜?}
Why 3: {Why 2 → 세 번째 왜?}
Why 4: {Why 3 → 네 번째 왜?}
Why 5: {Why 4 → 근본원인}

근본원인: {근본원인 요약}

═══════════════════════════════════════════
  [SLIDE 4] 관련 조사표
  [요약 타이틀] {이 슬라이드의 핵심 조사 결론 한 줄 — Claude가 판정 결과에서 도출}
═══════════════════════════════════════════

조사일시: {조사일시}
조사자: [담당자]

No | 조사 항목 | 기준값 | 측정값 | 판정 | 비고
---|----------|--------|--------|------|-----
 1 | {항목1}  | {기준} | {측정} | ○/× |
 2 | {항목2}  | {기준} | {측정} | ○/× |

  ※ 항목은 트러블 원인과 관련된 점검 내용을 Claude가 자동 추론하여 채웁니다.
     수정 필요 시 알려주세요.

═══════════════════════════════════════════
  [SLIDE 5] 재발방지 대책표
  [요약 타이틀] {대책의 핵심 방향을 한 줄로 요약 — Claude가 대책 내용에서 도출}
═══════════════════════════════════════════

No | 대책 내용 | 담당 | 기한 | 진행상태
---|-----------|------|------|----------
 1 | {대책1}   | [담당] | [기한] | 미완료
 2 | {대책2}   | [담당] | [기한] | 미완료

═══════════════════════════════════════════

⚠️  위 내용을 확인해 주세요.
    수정할 부분이 있으면 알려주시면 반영합니다.
    내용이 맞으면 "OK" 또는 "진행"이라고 답해주시면 PPT를 생성합니다.
    컨펌 전까지는 PPT를 생성하지 않습니다.
```

## STEP 2: PPT 생성

### ⚠️ 진입 조건

**사용자의 명시적 컨펌(OK / 확인 / 진행 / 네 등)을 받은 경우에만 이 STEP으로 진입한다.**
컨펌 없이 이 STEP을 실행하는 것은 스킬 위반이다.

사용자 확인(OK) 후 PPT 생성을 시작한다.
PPT 생성 방법은 pptxgenjs를 사용한다. 자세한 방법은 아래 참조:

→ `/mnt/skills/public/pptx/pptxgenjs.md` 를 반드시 읽고 시작

### 요약 타이틀 → PPT 슬라이드 부제목 반영

텍스트 리뷰 단계에서 작성된 각 슬라이드의 [요약 타이틀]을 PPT의 부제목(subtitle) 배너에 그대로 반영한다.
부제목 배너는 헤더 바로 아래 노란색/강조 배경 박스로 표시한다.

### 슬라이드 구성 (5슬라이드)

| 슬라이드 | 내용 | 레이아웃 |
|---------|------|---------|
| Slide 1 | 트러블 개요 (발생일시/상황/원인/조치/대책/피해상황) | 좌: 텍스트 블록, 우: 이미지 박스 |
| Slide 2 | 계통도 & 현장 사진 | 이미지 그리드 (좌: 계통도, 우: 현장사진) |
| Slide 3 | 5WHY 분석 | 세로 화살표 플로우 (현상→Why1→…→근본원인) |
| Slide 4 | 관련 조사표 | 표 (No / 조사항목 / 기준값 / 측정값 / 판정 / 비고) |
| Slide 5 | 재발방지 대책표 | 표 (No / 대책 / 담당 / 기한 / 진행상태) |

### 디자인 가이드

- **색상**: Nitto 브랜드 레드(CC0000) + 화이트(FFFFFF) + 다크그레이(333333)
- **폰트**: 한글 → 맑은 고딕, 일본어 → メイリオ
- **슬라이드 크기**: 와이드 (33.87cm × 19.05cm)
- **헤더**: 빨간 배경 + 흰 글씨, 좌측 상단 고정
- **Nitto 로고 텍스트**: 우측 하단 "Nitto / Innovation for Customers" (실제 로고 파일 없을 시 텍스트로 대체)
- **Confidential 표시**: 좌측 하단 빨간 박스 + 흰 글씨

### 계통도 & 현장 사진 슬라이드 (Slide 2)

- 사용자가 이미지를 제공한 경우: 해당 이미지를 삽입
- 이미지가 없는 경우: 회색 점선 박스 + "계통도 삽입" / "現場写真挿入" 텍스트로 플레이스홀더 표시
- 레이아웃: 좌측 60% 계통도(HMI 화면 등) + 우측 40% 현장 사진 1~2장
- 이미지 캡션을 각 이미지 하단에 추가

### 5WHY 슬라이드 구성 (Slide 3)

- 세로 플로우 다이어그램 형식
- 박스: 현상 → Why1 → Why2 → Why3 → Why4 → Why5(근본원인)
- 화살표로 연결
- 근본원인 박스는 빨간 강조색 적용

### 관련 조사표 슬라이드 구성 (Slide 4)

트러블 원인과 연관된 점검/측정 항목을 표로 구성. Claude가 원인 내용을 분석해 항목을 자동 추론한다.

| 컬럼 | 너비 비율 |
|------|---------|
| No | 5% |
| 조사 항목 / 調査項目 | 35% |
| 기준값 / 基準値 | 15% |
| 측정값 / 測定値 | 15% |
| 판정 / 判定 | 10% |
| 비고 / 備考 | 20% |

- 헤더 행: 빨간 배경 흰 글씨
- 데이터 행: 흰/연회색 교대 (zebra stripe)
- 조사일시 / 조사자 정보를 표 상단에 표시

### 재발방지 대책표 슬라이드 구성 (Slide 5)

| 컬럼 | 너비 비율 |
|------|---------|
| No | 5% |
| 대책 내용 / 対策内容 | 45% |
| 담당 / 担당 | 15% |
| 기한 / 期限 | 15% |
| 진행상태 / 進捗 | 20% |

- 헤더 행: 빨간 배경 흰 글씨
- 데이터 행: 흰/연회색 교대 (zebra stripe)

## STEP 3: QA

PPT 생성 후 반드시 QA를 수행한다.

```bash
# 텍스트 추출 확인
pip install "markitdown[pptx]" --break-system-packages -q
python -m markitdown output.pptx

# 이미지 변환 후 시각 검수
python /mnt/skills/public/pptx/scripts/office/soffice.py --headless --convert-to pdf output.pptx
pdftoppm -jpeg -r 150 output.pdf slide
ls -1 "$PWD"/slide-*.jpg
```

시각 QA 체크리스트:
- [ ] 텍스트가 박스 밖으로 넘치지 않는가
- [ ] 한글/일본어 폰트가 깨지지 않는가
- [ ] 빨간 헤더 색상이 정확한가
- [ ] 5WHY 화살표 플로우가 올바르게 연결되는가
- [ ] 대책표 열 너비가 적절한가
- [ ] Confidential / 페이지 번호가 모든 슬라이드에 있는가
