# 품의 패키지 통합 스킬 (pumui-package-generator)

## 개요

KORENO 공사 품의 전체 업무 플로우를 하나의 스킬로 커버한다.
각 Phase는 기존 스킬(pumeuiseo / gyeonsik-syangseo / estimate-comparison)을 내부에서 호출·통합한다.

```
공사 정보 입력
    │
    ├─ Phase 1+2 (동시 출력) ─────────────────────── 필수
    │   ├── 품의서 슬라이드 구성 텍스트 초안
    │   └── 호렌소 사전 진단
    │
    ├─ Phase 3: 견적사양서 (텍스트 → 선택적 DOC) ── 필수
    │           ↓ 업체 견적 수령
    ├─ Phase 4: 견적검토서 (XLSX) ─────────────────── 필수
    │
    ├─ Phase 5: 품의서 완성본 (PPT) ────────────────── 필수
    │
    ├─ Phase 6: JSA ─────────────────────────────── 선택 (필요 시 호출)
    ├─ Phase 7: 완료보고서 ─────────────────────────── 선택 (필요 시 호출)
    └─ Phase 8: 공사일보 ───────────────────────────── 선택 (필요 시 호출)
```

---

## 실행 전 정보 수집

사용자에게 아래 항목 확인. **불명확한 항목만** 역질문 (최대 3개):

| 항목 | 설명 | 필수 |
|---|---|---|
| 공사명 | 품의서·견적사양서 제목 | ✓ |
| 공사 내용 | 작업 범위·방법 요약 | ✓ |
| 공사 위치 | 건물·구역·층 | ✓ |
| 배경·이유 | 왜 필요한지 | ✓ |
| 예산 구분 | 경비/통상/전략 | ✓ |
| 예상 금액 | 대략 규모 | 권장 |
| 업체명 | 견적 대상 업체 1~2개 | 권장 |
| 스케줄 | 공사 희망 시기 | 권장 |
| 첨부파일 | 사진·견적서·기존 PPT | 있으면 자동 추출 |

> 첨부파일이 있으면 내용을 최대한 추출해 자동 채운다.

### Phase 2 출력 형식 — 호렌소 사전 진단

Phase 1 바로 아래 이어서 출력한다:

```
═══════════════════════════════════════════════════
  호렌소(I-SAFE) 사전 진단
═══════════════════════════════════════════════════

【a. 공사 주요 검토사항】
(공사 내용 분석 후 자동 도출. 해당 항목만 출력)
  ▶ 고소작업 포함 → 안전대·안전블럭 필요, 작업발판 계획 확인
  ▶ 계획정지 필요 → 해당 설비 정지 일정 사전 확인
  ▶ 방화구획 관통 → 방화댐퍼·방화처리 필요
  ▶ 인허가 선행 → 통합환경허가 P&ID 변경 확인
  ▶ 위험물 배관 접속 → NaOH·NG 계통 격리 LOTO 확인
  ▶ 스팀 계통 → 드레인 유도 위치, 플랜지 가스켓 재질 확인
  ▶ 전기공사 → 방폭 여부, 접지 방식 확인

【b. 추가 확인이 필요한 사항】
(공사 내용 분석 후 자동 도출. 해당 항목만 출력)
  ▷ 드레인 유도 위치 확인 필요
  ▷ 하부 배관 간섭 여부 확인 필요
  ▷ 재질 결정 필요 (SUS316L vs SUS304)
  ▷ 계통도·P&ID 확인 필요
  ▷ 발주 완료 여부 확인

【c. 호렌소 관련 확인사항】

  a) 예상 호렌소 등급 (금액 기준 자동 판단)
     100만원 미만     → 파트장(백두한) 결재
     500만원 이하     → 팀장(최세환) 결재
     5,000만원 이하   → 총괄부장(오타 상무)
     1억원 이하       → 대표이사(이배원 사장) + 코센 + 이사회
     1억~3억원        → 대표이사 + 코센 + 투자심의회
     → 이 건: [금액 기반 자동 판단 결과]

  b) 방화 NIS 체크 항목 (공사 내용 기반 자동 도출)
     (해당 NIS 번호 자동 출력)
     예: 전기공사 → S-0103 / S-0105 / S-0107
         배관공사 → S-0101 / S-0102
         덕트공사 → T-0402 / T-0404
         고소작업 → T-0502

  c) 안전설계 비대상 여부
     □ 비대상 — 단순 노후화 갱신 / 소모품 교체
     □ 대상   — 신규 설비 / 공정 변경 / 위험물 계통
     ※ 금액·위험도 관계없이 "단순 노후화 갱신"이면 형식상 비대상
       → 위험도 높은 건은 자체 RA 추가 실시 권장

  d) 간이체크시트 해당 여부
     아래 14개 항목 중 해당 항목 자동 표시:
     □ 기계적 위험원 (끼임·협착)     □ 절단 위험원
     □ 감전 위험원 (AC 25V↑)        □ 화재·폭발 위험원
     □ 열적 위험원 (고온 스팀·배관)  □ 파열·분출 위험원
     □ 추락 위험 (고소작업)          □ 낙하 위험
     □ 충돌 위험                     □ 소음 (85dB↑)
     □ 방사 위험원                   □ 산소결핍·질식
     □ 중량물 (5kg↑)                □ 유해 물질 (NaOH·NG·KI 등)
     → 1개 이상 해당 시: 간이체크시트 작성 필요

  e) 계획 RA 3자 회담 필요 여부
     □ 불필요 (비대상)
     □ 필요 → 참석자: 실행(구유성) + 유관(해당 파트) + 환경안전(담당자)

【제출 서류 최종 체크리스트】
  ✅ 상세설명자료 (품의자료)    — 항상 필수
  ✅ 견적서                     — 항상 필수
  ✅ 방화NIS 체크리스트          — 항상 필수
  □  안전설계 비대상 판단표      — 비대상 시 필수
  □  간이체크시트               — 대상 시 필수
  □  계획RA 3자 회담 회의록     — 대상 시 필수
═══════════════════════════════════════════════════
```

## Phase 4 — 견적검토서

**기반 스킬:** `estimate-comparison-generator` 로직을 통합

### 입력 요청

```
견적검토서 작성을 위해 아래 정보를 입력해주세요:

문서번호: (예: KEKIO-002)
작성일:
건명: (공사명)
예산구분: □전략 / □통상 / □경비
발표자:
납기:
지불조건:

업체 1 (좌측): ___
업체 2 (우측): ___

항목별 견적:
구분 | 내역 | 업체1 수량 | 단위 | 단가 | 업체2 수량 | 단위 | 단가 | 검토코멘트
```

### XLSX 구조 (openpyxl)

```python
# 컬럼 구성 (B~L)
# B: 구분 | C: 내역 | D: 수량1 | E: 단위1 | F: 단가1 | G: 금액1(=D*F)
# H: 수량2 | I: 단위2 | J: 단가2 | K: 금액2(=H*J) | L: 검토내용

# 헤더 (Row 1~12): 문서번호·건명·작성일·참석인원·예산구분·발표자·납기·지불조건·확인자·업체명
# 본문: 구분별 소계(SUM 수식) + 재료비/노무비 항목
# 합계: 간접비 → 합계 → Discount → 발주금액
# 의견란: 견적 검토 의견 + 참석자 의견

# 색상: 낮은 금액 셀 → 연녹색(C6EFCE) 하이라이트
# 수식: G열 = D*F, K열 = H*J, 소계 = SUM(), 합계 자동
```

### 견적 검토 의견 작성 규칙

```
[건명]의 건으로 (업체1)、(업체2) 양사에 대해 비교견적을 실시하였음.

양사 전체 금액을 비교하면, (업체1) XXX만원 / (업체2) XXX만원으로
(저렴한 업체)가 약 X.X%(약 XXX만원) 낮게 책정됨.

공사금액·대응력·당사 시공 경험 등을 종합 검토하여 (선정 업체)를 선정.
(선정 이유 구체적으로 기술)

단, (조정 가능 항목)에서 공수 조정 가능하다고 판단됨.
견적 전체 금액(XXXx만원)으로부터 약 XXX만원(X.XX%) 절충 가능하다고 판단됨.
```

**작성 원칙:**
- 업체명은 반드시 (괄호) 표기
- 금액 차이: 절대금액(원) + 비율(%) 모두 기재
- 선정 근거: 금액 외 요소(경험·대응력·안전인식 등) 포함
- 절충 가능 금액·근거가 있으면 마지막에 기재
- 전체 3~5문장 이내 간결하게

```bash
# 파일명: yymmddhhmm_공사명_견적검토서.xlsx
cp /home/claude/output.xlsx "/mnt/user-data/outputs/$(date +%y%m%d%H%M)_견적검토서.xlsx"
```

## Phase 5 — 품의서 완성본 (PPT)

**기반 스킬:** `pumeuiseo-generator` 로직을 통합

Phase 1 초안 + Phase 4 견적 데이터 통합 → PPT 생성

### 트리거
사용자가 아래 중 하나 입력 시 실행:
- "품의서 PPT 만들어줘" / "완성본 만들어줘" / "견적 반영해서 PPT 출력해줘"

### 슬라이드 구성 (9슬라이드 고정)

| # | 슬라이드 | 핵심 내용 |
|---|---|---|
| 0 | 표지 | 제목·회의체·승인자·보고자·날짜·부의사항 |
| 1 | 서머리 | 과제·원인·해결책·효과·코스트·스케줄 |
| 2 | 공사개요 | 배경·범위·금액·사진·예산구분 체크박스 |
| 3 | 과제 | 문제점 표 + 리스크 + 사진 |
| 4 | 원인 | 원인 분석 + 법규·기술 근거 |
| 5 | 해결책 | 해결 방안 + 개념도 + 진행 순서 |
| 6 | 효과 | Before/After + 기대효과 |
| 7 | 견적 검토 | Phase 4 견적검토서 데이터 연동 |
| 8 | 스케줄 | 월별 간트 차트 (2~3개월) |

### 실행 절차

```bash
# Step 1: pumeuiseo SKILL.md 읽기 → 디자인 규칙 확인
cat /mnt/skills/user/pumeuiseo-generator/SKILL.md

# Step 2: pptxgenjs 스크립트 작성
# /home/claude/pumeuiseo_output.js

# Step 3: 실행 및 QA
node /home/claude/pumeuiseo_output.js

## 전체 흐름 제어 규칙

### 단계 진행

1. **Phase 1+2**: 정보 수집 후 즉시 동시 출력 → 사용자 확인 대기
2. **Phase 3**: Phase 1+2 확인 완료 후 출력 → DOC 여부 확인

"완료보고서"   → Phase 7 단독
"공사일보"     → Phase 8 단독
```

## Phase 6 — JSA (작업안전분석)

> **상태: 참고자료 추후 첨부 예정 — 현재 공란 스텁(stub)**  
> 참고자료 수령 후 본 섹션을 완성한다.

### 트리거
- "JSA 만들어줘" / "작업안전분석" / "JSA 필요해"

### 현재 동작
사용자가 요청 시 아래 안내 출력:

```
═══════════════════════════════════════════════════
  JSA (작업안전분석) — 준비 중
═══════════════════════════════════════════════════

JSA 스킬은 참고자료 검토 후 완성 예정입니다.

현재 입력 가능한 정보:
  공사명: (공사명)
  작업 단계: (공사 내용에서 자동 추출)
  위험 요인: (Phase 2 간이체크시트 결과 자동 연동)

완성본 스킬 등록 후 자동 생성됩니다.
```

### 향후 구성 (참고자료 수령 후 완성)
```
JSA 양식 구조 (예정):
  No | 작업 단계 | 위험 요인 | 위험 등급 | 안전 대책 | 담당자
  ───────────────────────────────────────────────────────
  Phase 2의 간이체크시트 14항목 → 자동 연동 예정
  출력 형식: XLSX 또는 DOC (결정 후 반영)
```

## Phase 7 — 완료보고서

> Phase 3 견적사양서 + 실제 공사 결과를 기반으로 작성

### 트리거
- "완료보고서 만들어줘" / "공사 완료 보고" / "완료보고"

### 입력 요청

```
완료보고서 작성을 위해 아래 정보를 입력해주세요:

[견적사양서에서 자동 인계되는 항목]
  공사명: (자동)
  위치: (자동)
  업체명: (자동)
  공사 내용: (자동)

[추가 입력 필요]
  실제 공사 기간: 착공 YYYY.MM.DD ~ 준공 YYYY.MM.DD
  실제 금액: ___원 (견적 대비 증감 여부)
  공사 결과 사진: (첨부)
  특이사항: (공사 중 발생한 이슈, 변경사항 등)
  검수자: (확인한 담당자)
  준공서류 목록: □사진 □검사성적서 □AS보증서 □시운전기록 □기타
```

### 출력 형식 (텍스트 → DOC)

```
═══════════════════════════════════════════════════
  공사 완료보고서
═══════════════════════════════════════════════════

1. 공사 개요
   공사명: (공사명)
   위치: (건물·구역·층)
   업체명: (업체명)
   공사 기간: YYYY.MM.DD ~ YYYY.MM.DD (XX일)

2. 공사 내용
   ① ...
   ② ...
   ③ ...

3. 공사 결과
   금액: 견적 XX원 → 실제 XX원 (차이: +/-XX원)
   특이사항: ...

4. 준공 사진
   [사진 플레이스홀더 — 첨부 이미지 자동 삽입]

5. 준공서류 체크리스트
   □ 공사 완료 사진
   □ 검사성적서
   □ AS 보증서
   □ 시운전 기록
   □ 기타: ___

6. 확인
   검수자: (이름)    확인일: YYYY.MM.DD
───────────────────────────────────────────────────
DOC 파일로 출력할까요?
───────────────────────────────────────────────────
```
