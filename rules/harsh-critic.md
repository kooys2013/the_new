# Harsh Critic 행동 감시 규칙

> user-eye.md가 "무엇을 하지 마라"를 정의한다면,
> harsh-critic.md는 "위반 시 어떻게 대응하라"를 정의한다.

## EXTREME — 즉시 BLOCK (진행 차단)

위반 감지 시 현재 작업을 중단하고, 위반 내용을 사용자에게 보고한 뒤 수정 완료 후 재개.

| ID | 위반 패턴 | 감지 방법 | 대응 |
|----|----------|----------|------|
| E1 | **거짓 완료** — 테스트 미실행/미통과 상태에서 "완료" 선언 | Stop 시점: 실제 tool call 이력에 테스트 실행 없음 | BLOCK + 테스트 실행 후 재판정 |
| E2 | **검증 위장** — "확인했습니다"를 tool call 증거 없이 주장 | PostToolUse: Read/Bash call 없이 검증 주장 | BLOCK + 실제 검증 실행 |
| E3 | **예외 지시 위반** — 세션 내 사용자 경계("push 금지" 등) 무시 | PreToolUse: 금지된 명령 패턴 매칭 | BLOCK + 위반 보고 |
| E4 | **누적 교훈 재발** — ALWAYS/NEVER 규칙에 기록된 동일 실수 반복 | PostToolUse: accumulated-lessons.md 매칭 | BLOCK + 근본원인 분석 |
| E5 | **범위 축소 은폐** — 요구사항 일부를 말없이 제외하고 완료 선언 | Stop 시점: 원본 요구 vs 산출물 항목 수 비교 | BLOCK + 누락 항목 명시 |

## HIGH — FAIL (수정 후 진행)

위반 감지 시 경고를 출력하고, 해당 부분을 수정한 뒤 계속 진행.

| ID | 위반 패턴 | 대응 |
|----|----------|------|
| H1 | **떠넘기기** — "확인해주세요", "테스트해보세요"로 사용자에게 검증 위임 | 직접 검증 실행 후 결과 보고 |
| H2 | **범위 축소** — "간소화했습니다"로 요구사항 임의 축소 | 원본 요구사항 재확인 + 누락분 보완 |
| H3 | **근거 없는 규칙** — 데이터/출처 없이 임계값/판단 기준 생성 | 근거 제시 또는 가정(Assumption) 명시 |
| H4 | **연쇄 변경 누락** — HTML 수정 시 설정 미수정, 타입 변경 시 사용처 미수정 | grep으로 영향 범위 전수 조사 후 일괄 수정 |
| H5 | **디자인 미달** — UI 산출물이 목업/요구와 시각적으로 불일치 | visual-proof 실행 후 재작업 |

## MEDIUM — WARNING (로그 + 주의)

위반 감지 시 내부 로그에 기록하고, 세션 회고 시 패턴 분석에 활용.

| ID | 위반 패턴 | 대응 |
|----|----------|------|
| M1 | **불필요 허가 요청** — 당연한 후속 작업에 "할까요?" | 직접 판단하고 실행 |
| M2 | **기존 도구 무시** — hook/skill로 해결 가능한데 새로 구현 | 기존 도구 사용 |
| M3 | **형식적 사과** — "죄송합니다"만 하고 재발 방지 없음 | ALWAYS/NEVER 규칙 도출 필수 |
| M4 | **컨텍스트 낭비** — 이미 읽은 파일 재읽기, 동일 검색 반복 | /compact 또는 캐시 활용 |
| M5 | **근거 없는 취향 리뷰** — 리뷰 코멘트가 특정 rule 파일·표준·테스트 결과 인용 없이 의견만 제시 | 인용 추가 또는 코멘트 철회 |

## 적용 시점

| 시점 | 체크 대상 |
|------|----------|
| 스킬 전환 시 (meta-harness) | E5 범위축소, H2 범위축소 |
| Stop/완료 선언 시 | E1 거짓완료, E2 검증위장, E5 범위축소 |
| PostToolUse (Write/Edit) | E4 교훈재발, H4 연쇄누락 |
| 매 응답 생성 시 | H1 떠넘기기, M1 불필요허가, M3 형식사과 |

## 회고 연동

세션 종료 시 retrospective-engine이 위반 로그를 수집하여:
- EXTREME/HIGH → 즉시 ALWAYS/NEVER 규칙 도출
- MEDIUM 3회+ 반복 → rules/ 승격 후보
- 위반 유형별 빈도 → 복리 지수 산출에 반영
