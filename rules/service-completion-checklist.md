# 서비스 완료 문제 (Service Completion Problem) 체크리스트

> "LLM은 5가지 구조적 이유로 작업을 완료하지 못한다.
>  이름을 붙여야 세고, 세야 줄일 수 있다."

## 5대 구조적 실패 유형

| ID | 유형 | 증상 | 방어 로직 |
|----|------|------|----------|
| SCP-1 | **조기 종료** (Premature Termination) | 부분 구현, 미테스트 코드, 엣지케이스 누락 | Stop 시 요구사항 vs 산출물 1:1 매핑 검증 |
| SCP-2 | **컨텍스트 열화** (Context Degradation) | 초반 결정 모순, 요구사항 망각, 스타일 불일치 | 30분 /compact + 핵심 제약 재주입 |
| SCP-3 | **복잡도 회피** (Complexity Avoidance) | 요구사항 암묵적 축소, "간소화" 언급, 엣지케이스 연기 | 원본 요구사항 diff + 축소 시 사용자 명시적 승인 |
| SCP-4 | **책임 분산** (Responsibility Diffusion) | 과도한 질문, "남겨두겠습니다" 표현, 모호한 TODO 목록 | 시작 가능하면 완료 가능. 판단 가능한 것은 직접 결정 |
| SCP-5 | **검증 연극** (Verification Theater) | 미실행 테스트 통과 주장, 미읽기 파일 리뷰 주장 | tool call 이력에 실행 증거 필수 |

## 작업 시작 시 (Pre-flight)

복잡 작업(2턴 이상 예상) 시작 시 아래를 자문:
- [ ] 요구사항을 번호 매긴 체크리스트로 분해했는가?
- [ ] 각 항목의 완료 판정 기준이 수치/자동검증 가능한가?
- [ ] 컨텍스트 60% 초과 예상 시 /compact 시점을 계획했는가?

## 작업 완료 시 (Post-flight)

Stop/완료 선언 전 아래 5개 질문에 모두 "예":
1. **SCP-1**: 요구사항 체크리스트 전 항목 완료인가?
2. **SCP-2**: 초반 결정과 최종 산출물이 일관되는가?
3. **SCP-3**: 원본 요구 대비 축소된 부분이 없는가? (있으면 명시적 승인 필요)
4. **SCP-4**: 사용자에게 "확인해주세요" 없이 직접 검증했는가?
5. **SCP-5**: 테스트/빌드/린트 실행의 tool call 증거가 있는가?

## 회고 분류 태그

retrospective-engine 실행 시 실패/개선 항목에 SCP 태그 부여:
```
[SCP-1] 조기종료: 3개 요구사항 중 2개만 구현
[SCP-3] 복잡도회피: 에러 처리를 "추후 추가"로 연기
```
SCP별 빈도 추적 → 가장 빈번한 유형에 방어 강화
